## 版本管理策略

### 版本号格式

采用语义化版本号格式：`MAJOR.MINOR.PATCH`

- **MAJOR**：重大变更，不向后兼容的API变更
- **MINOR**：新增功能，向后兼容
- **PATCH**： bug修复，向后兼容

### 发布流程

1. **开发阶段**：在开发分支上进行功能开发和bug修复
2. **测试阶段**：在测试环境中进行集成测试和性能测试
3. **发布准备**：更新版本号，编写发布说明
4. **发布阶段**：将代码合并到主分支，构建发布包
5. **部署阶段**：部署到生产环境，进行上线测试
6. **维护阶段**：收集用户反馈，修复bug，准备下一个版本

### 分支管理

- **main**：主分支，包含稳定的生产代码
- **develop**：开发分支，包含最新的开发代码
- **feature/**：功能分支，用于开发新功能
- **bugfix/**：bug修复分支，用于修复bug
- **release/**：发布分支，用于准备发布版本

### 版本发布流程

1. 从develop分支创建release分支
2. 在release分支上进行版本号更新和发布准备
3. 进行最终测试和验证
4. 将release分支合并到main分支
5. 从main分支创建tag
6. 发布版本
7. 将release分支合并回develop分支
