#PeaBSS 开发文档

#1. 系统架构

PeaBSS（豌豆易联业务服务支撑系统）是一个基于 Go 语言开发的环保物联网平台，主要用于环保监测设备的数据采集、存储、分析和可视化。

#1.1 核心组件

  • BSS-Core: 核心业务逻辑处理
  • HJ212-Service: HJ212 环保监测协议服务
  • Web-Server: Web 管理界面服务
  • MCP-Server: 模型上下文协议服务（AI 集成）
  • SSE-Stream: 实时数据推送服务

#1.2 数据存储

  • PostgreSQL: 主数据库
  • TimescaleDB: 时间序列数据存储
  • FreeCache: 内存缓存

#1.3 技术栈

  • 后端: Go 1.21
  • 前端: Webix、ECharts、Leaflet
  • 数据库: PostgreSQL + TimescaleDB
  • 协议支持: HJ212、MQTT、Modbus
  • 部署: Docker、GitHub Actions

#2. 快速开始

#2.1 环境要求

  • Go 1.21 或更高版本
  • PostgreSQL 14 或更高版本（含 TimescaleDB 插件）
  • 操作系统: Linux、Windows

#2.2 安装步骤

#2.2.1 数据库准备

使用 Docker 快速部署 PostgreSQL + TimescaleDB:

# 创建本地目录
mkdir -p /database/pgdb
chmod 700 /database/pgdb
chown -R 1000:1000 /database/pgdb

# 创建 docker-compose.yml 文件
cat > /database/docker-compose.yml << EOF
version: '3'
services:
  postgres:
    image: timescale/timescaledb:latest-pg14
    ports:
      - "5432:5432"
    environment:
      POSTGRES_DB: peaiot
      POSTGRES_USER: peaiot
      POSTGRES_PASSWORD: peaiot
    volumes:
      - /database/pgdb:/var/lib/postgresql/data
EOF

# 启动容器
cd /database
docker-compose up -d

#3. 核心功能

#3.1 设备管理

  • 设备注册: 自动注册和手动添加设备
  • 设备状态监控: 实时监控设备在线状态
  • 设备配置: 支持设备参数配置
  • 设备分组: 按组织和项目分组管理

#3.2 数据采集

  • HJ212 协议: 支持环保监测设备标准协议
  • MQTT 协议: 支持 MQTT 设备接入
  • Modbus 协议: 支持 Modbus TCP/RTU 设备接入
  • 实时数据: 实时采集和处理设备数据

#3.3 数据存储

  • 时间序列数据: 使用 TimescaleDB 存储时序数据
  • 历史数据: 支持长期历史数据存储
  • 数据压缩: 自动数据压缩和清理

#3.4 数据可视化

  • GIS 大屏: 地理信息系统集成的监控大屏
  • AI BI 大屏: 基于 AI 分析的业务智能大屏
  • 实时数据图表: 实时数据趋势和分析
  • 历史数据报表: 历史数据查询和报表

#3.5 告警管理

  • 阈值告警: 基于阈值的设备告警
  • 多渠道通知: 短信、微信等告警通知
  • 告警历史: 告警记录和历史查询

#3.6 AI 集成

  • MCP 服务: 模型上下文协议服务
  • 智能诊断: 设备健康诊断和故障预测
  • 数据分析: 基于 AI 的数据分析和预测
  • 智能报表: 自动生成智能分析报表

peabss/
├── assets/         # 前端资源
├── common/         # 公共库
│   ├── e212/       # HJ212 协议实现
│   ├── modbus/     # Modbus 协议实现

#4.2 开发流程

  1. 环境准备: 安装 Go 1.21 和 PostgreSQL
  2. 代码克隆: git clone <repository-url>
  3. 依赖安装: go mod tidy
  4. 配置文件: 创建 peabss.yml 配置文件
  5. 数据库初始化: ./peabss -initdb
  6. 开发运行: go run main.go
  7. 测试: go test ./...
  8. 构建: go build -o peabss main.go

#4.3 代码规范

  • 使用 Go 标准代码风格
  • 遵循 Go 最佳实践
  • 使用 golangci-lint 进行代码检查
  • 编写单元测试

#6. 部署指南

#6.1 Docker 部署

# 创建 Dockerfile
cat > Dockerfile << EOF
FROM golang:1.21-alpine as builder
WORKDIR /app
COPY . .
RUN go mod tidy
RUN CGO_ENABLED=0 GOOS=linux GOARCH=amd64 go build -a -ldflags '-s -w -extldflags "-static"' -o peabss main.go

FROM alpine:latest
WORKDIR /app
COPY --from=builder /app/peabss .
COPY --from=builder /app/assets ./assets
EXPOSE 1870 1872 1875
CMD ["./peabss"]
EOF

# 构建镜像
docker build -t peabss:latest .

# 运行容器
docker run -d -p 1870:1870 -p 1872:1872 -p 1875:1875 --name peabss peabss:latest

#6.2 系统服务部署

# 安装为系统服务
./peabss -install

# 启动服务
systemctl start peabss

# 停止服务
systemctl stop peabss

# 查看状态
systemctl status peabss

#8. 扩展指南

#8.1 新增设备协议

  1. 在 common/ 目录下创建新的协议实现
  2. 在 service/ 目录下添加对应的服务
  3. 在设备管理中添加新的设备类型

#8.2 新增功能模块

  1. 在 controllers/ 目录下创建新的控制器
  2. 在 service/ 目录下添加对应的服务
  3. 在菜单管理中添加新的菜单项

#8.3 集成第三方服务

  1. 在 common/ 目录下添加第三方服务客户端
  2. 在 service/ 目录下集成服务逻辑
  3. 在配置文件中添加相关配置

#9. 安全指南

#9.1 认证与授权

  • JWT 认证: 所有 API 接口使用 JWT 认证
  • 角色权限: 基于角色的权限控制
  • 密码加密: 密码使用 SHA256 加密存储

#9.2 数据安全

  • 数据传输: 使用 HTTPS 加密传输
  • 数据存储: 敏感数据加密存储
  • 访问控制: 基于组织的访问控制

#9.3 系统安全

  • 防火墙: 配置适当的防火墙规则
  • 定期更新: 定期更新系统和依赖
  • 安全审计: 定期进行安全审计

#10. 版本管理

#10.1 版本规范

采用语义化版本规范：MAJOR.MINOR.PATCH

  • MAJOR: 重大变更，不兼容的 API 更改
  • MINOR: 新功能，向后兼容
  • PATCH: bug 修复，向后兼容

#10.2 版本历史

版本日期描述
v3.0.812026-04-15基础版本
v3.0.822026-04-15新增 AI 对话功能和 MCP 工具测试页面

#10.3 发布流程

  1. 代码开发: 在 develop 分支开发
  2. 测试: 运行测试和 lint 检查
  3. 合并: 合并到 main 分支
  4. 标签: 创建版本标签
  5. 发布: GitHub Actions 自动构建和发布

#11. 故障处理

#11.1 常见问题

问题原因解决方案
设备离线网络连接问题检查网络连接和设备状态
数据异常设备故障或传感器问题检查设备和传感器
