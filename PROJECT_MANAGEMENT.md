# PEAIOT 多站点工程项目管理纪要 (V0.1.0)

> 本文档基于信息系统项目管理师（高项）模型裁剪，保留最小化敏捷交付步骤，作为项目的开发标准与基线管理文档。

## 1. 范围与基线管理 (Scope & Baseline)
- **项目目标**：交付高可用、纯静态的 PEAIOT 企业主站 (`peaiot.cn`) 与企业文档中心 (`doc.rry.net`)。
- **V0.1.0 交付物基线**：
  - 基于 Rspress 构建的多站点物理隔离架构（`www_src` 与 `chromatography-rspress-docs` 彻底解耦）。
  - `sites.json` 自动化站点集中注册与配置管理。
  - 企业主站新增“软件定制开发”业务模块与极简商业文案。
  - 企业文档中心暗黑模式深度适配与框架去痕（Logo 徽章化 CSS 隔离、隐藏原生搜索组件）。
  - 基于宝塔 API 的一键自动化部署脚本 (`deploy_to_bt.js`) 与 AI 专属部署技能。

## 2. 进度与质量控制 (Schedule & Quality)
- **质量红线管理**：严格执行“本地化原则”，绝对禁止任何第三方外部服务与资源依赖（图床、CDN等），确保企业数据与物理级网络安全。
- **架构风险应对**：识别并修复了早期单目录构建导致的 CSS 样式跨站污染问题，重构为目录级物理隔离方案。

## 3. 配置与发布管理 (Configuration & Release)
- **版本控制**：统一采用 Git 进行配置管理，当前核心里程碑打上 `v0.1.0` 标签。
- **持续交付 (CD)**：打通打包、压缩、API 鉴权上传、解压覆盖的全自动流水线，实现“AI 一句话即发布”。

## 4. 下一步规划 (Next Steps)
- 收集线上反馈，迭代核心业务模块的内容与详情页。
- 逐步丰富和完善各硬件仪器、软件平台的详细技术实施文档。
