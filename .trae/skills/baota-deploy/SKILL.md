# 宝塔自动化部署技能 (Baota Deploy)

这是一个高度自动化的网站部署技能。当用户要求“更新站点”、“发布网站到宝塔”、“部署项目”时，请作为 AI 助手全自动执行以下流程，**不要只给用户步骤，而是直接执行完毕并汇报结果**。

## 核心前置要求 (环境改造)
项目已重构为“档案化管理”，所有站点配置存放在 `sites.json` 中，部署脚本 `deploy_to_bt.js` 现已支持从 `sites.json` 读取指定站点的部署参数（如所使用的宝塔面板前缀，目标输出目录等），并配合 `.env` 中的凭证自动部署。

在执行部署前，请确保：
1. 目标站点在 `sites.json` 中有对应的配置。
2. `.env` 中有对应的环境变量（如 `BTA6_PANEL_URL`）。
3. 构建产物目录与站点配置的要求一致。

## 标准执行流程

在收到用户的部署请求后，严格按以下步骤依次执行：

### 1. 项目构建 (Build)
根据用户要求部署的站点，执行对应的构建命令。
- 部署 `peaiot.cn`，执行 `npm run build:www`
- 部署 `doc.rry.net`，执行 `npm run build:doc`
- 如果要求全部部署，执行 `npm run build:all`

等待命令执行完毕。

### 2. 执行部署脚本 (Deploy)
执行改造好的部署脚本，并传入目标站点名称。
> **注意**：脚本内部已经实现了自动打包压缩对应的 `outputDir` 目录，不需要你再手动调用 PowerShell 的 `Compress-Archive`。
示例命令：
```powershell
node deploy_to_bt.js <站点名称>
```
此脚本将自动通过宝塔 API 完成：打包目录 -> 新建/识别站点 -> 申请 SSL -> 上传对应 ZIP -> 覆盖解压。

### 3. 结果汇报 (Report)
所有命令执行完毕后，分析命令日志，向用户输出一份简短的成功报告（或失败的错误排查建议），不需要输出中间繁杂的哈希计算或命令执行过程。

## 异常处理预案
- **文件不存在报错**：如果是上传脚本执行报错，检查 `.env` 文件是否存在及凭证是否正确（切勿在对话中暴露 `.env` 内容）。
- **打包失败**：如果是 `Compress-Archive` 报错，检查构建目录名称是否与脚本期望的匹配。
